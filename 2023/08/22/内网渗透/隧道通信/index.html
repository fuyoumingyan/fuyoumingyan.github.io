隧道代理

隧道代理

基础知识

隧道技术：解决不出网协议上线问题（ 利用出网协议进行封装 ）

代理技术：解决网络通讯不通的问题（ 利用跳板机建立节点后续操作 ）

防火墙：

  1. 入站规则：外 => 内

  2. 出站规则：内 => 外

后门：

  1. 正向连接：靶机将 shell 放到本地，攻击机主动连接靶机 shell（ 攻击机 => 靶机 ，我主动联系，入站）
  2. 反向连接：靶机直接上 shell 发送给攻击机，攻击机监听本地端口 （ 靶机 => 攻击机，我等他联系，出站 ）

受防火墙影响时，也可以不使用隧道、代理技术，直接去关闭防火墙、修改出入站规则、修改规则白名单程序。但是这种操作需要有一个前提条件，那就是该用户有足够的权限。在没有域的内网环境中，只要该用户权限够高即可；但在存在域的内网环境中，域控是可以设置组策略防火墙同步，该策略只有域控才能关闭，所以直接干防火墙适用于以下两种情况：

  1. 无域内网环境
  2. 域控没有设置组策略防火墙同步

隧道技术

基础知识

思路：根据防火墙的入站或出站限制，选择正向或者反向的后门，在利用不同的协议实现通信。

  1. 入站无限制：正向连接
  2. 出站无限制：反向连接
  3. 入站、出站都有限制：反向连接 + 隧道技术（ 考虑限制内容 ）

隧道技术：向下兼容

如：当 TCP 协议被禁止，可尝试 ICMP 这种向下的协议。

模型 隧道技术
应用层 SSH、HTTP/S、DNS
传输层 TCP、UDP
网络层 IPv6、ICMP

判断协议是否可以使用：

协议 windows linux
ICMP ping ping
SSH ssh
DNS nslookup dig、nslookup
SMB 445 端口

ICMP隧道技术

ICMP 是依靠 ping 命令去和远端主机进行通信的，ICMP 隧道技术就是在通信过程中将 TCP/UDP 的数据都封装在 ICMP 的 ping 数据包中，从而绕过防火墙。

攻击过程：生成一个 lhost = 127.0.0.1 的反向后门，靶机在执行后门后，会将 TCP 数据先反弹到靶机本地，然后在靶机上使用 pingtunnel 将 TCP 的数据封装为 ICMP 数据进行出网。出网后的数据是 ICMP ，所以需要在攻击机上启动 pingtunnel 解除封装为 TCP 使用。

SSH隧道技术

SSH 隧道技术需要使用 Linux 主机且必须是 root 权限，且 SSH 隧道不涉及上线，类似于代理技术，使用 Linux 作为跳板机进行内网通讯。

主要分为：

  • 远程转发：将跳板机上访问的IP地址及端口流量转发至攻击机上的某个端口
  • 本地转发：将跳板机上访问的IP地址及端口流量转发至跳板机本地的某个端口
  • 动态转发：利用 ssh 隧道的正向 socks 代理把目标的整个内网代出来

ssh 参数说明：

  • -C：压缩传输，提高传输速度

  • -f：将SSH传输转入后台执行，不占用当前的Shell

  • -N：建立静默连接(建立了连接，但是看不到具体会话)

  • -g：允许远程主机连接本地用于转发的端口

  • -L：本地端口转发

  • -R：远程端口转发

  • -D：动态转发(SOCKS代理)

  • -P：指定SSH端口

正向连接

  • winServer 1 出网，可上线

在这种网络不能通讯的情况下，可利用代理技术来解决通信问题，即利用可控制的机器作为跳板机去控制不出网的机器。
